元件

本頁盤點全站實際引用中的 UI 元件，逐一列出使用情況與樣式規格，做為前端開發與設計協作的共同依據。每個元件依「讀碼 → 整理現況 → 提案 → 拍板」四步驟逐一定案：標「★ 待盤點」的尚未深入檢視；已有預覽圖的為現況記錄，規格同樣以定案為準。

新專案接入指南

所有資料匯流平台相關網站的新專案，接入設計系統的標準流程。本設計系統是唯一來源：token、共用元件、元件規格都以此為準，不得在各專案自行發明。完整步驟與程式碼片段見 repo 根目錄的 INTEGRATION.md。

元件來源三層制

層級標籤實體位置使用方式
shadcn 原生shadcn 原生各專案自己的 components/ui/shadcn CLI 安裝，不修改
shadcn + 有意修改有意修改各專案 components/ui/CLI 安裝後重放拍板過的修改 ( 各元件章有寫，檔內留註解 )
自訂自訂設計系統 repo 的 components/由套件 import，不得複製進專案

接入五步驟

1. 安裝套件：bun add github:zeroflare/nics-dcf-design-system，CSS 進入點 @import '@nics/design-tokens/tokens.css'，並加 @source 指到套件的 components/ ( Tailwind 預設不掃 node_modules )
2. shadcn 橋接變數指向 token：--primary = brand-500 · --destructive = red-500 · --ring = brand-300 · --border = stroke
3. 安裝認可的 shadcn 元件清單並重放 3 個有意修改 ( Dialog / Card / Progress )；明確不使用 alert、scroll-area、膠囊式 tabs
4. 自訂元件從套件 import ( 如 @nics/design-tokens/components/Tag.vue )，依賴契約：@ 別名指向 src、對應 shadcn 元件已安裝
5. 實作前先讀本頁對應元件的「已拍板規則」；需要新元件或新樣式 pattern，先回設計系統立章拍板，實體放設計系統的 components/
