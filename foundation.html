設計規範

本頁收錄 DCF 存取節點平台正式拍板的 Design Token 規範，包含顏色、文字、間距、圓角、陰影、頁面佈局，並附上狀態色票與審計清單作為現況參考。

顏色

Token 架構分三層：Base 層共 13 條純色階 ( 12 條各 10 階 50–900，neutral 多一階 0 純白，共 11 階 )，其中 8 條為核心、5 條僅供分類辨識；Function 層只引用 Base 的階數，不產生新色值；Component 層再收斂一次給實際元件用。需求書的甲方色一律錨定在各色階的 500 作為前景基準，已選取底色 #f0fdfa 自然落在 brand-50。

Layer 1 · Base

純色階，不帶語意

neutral · brand 之外，其餘 11 條依色相排成 red · orange · yellow · green · aqua · sky · blue · indigo · purple · pink · rose，其中 red / orange / yellow / green / blue / aqua 為核心、其餘 5 條僅供分類辨識。這一層只回答「有哪些顏色」，不回答「什麼時候用」

Layer 2 · Function

語意層，只引用 Base

分兩組：中性骨架 ( foreground-primary/secondary、stroke、background-surface/container ) 描述頁面本身；語意色 ( success/warning/danger/info ) 只有 foreground、background、stroke 三個 slot

Layer 3 · Component

元件層，只引用 Function

以實際元件為單位再收斂一次，例如 Tag 元件只有 background 跟 foreground 兩個 slot，指向 Function 層的語意 token

Function 層 · 中性骨架

描述頁面本身的中性色，不含任何語意。foreground 分 primary / secondary / tertiary 三階涵蓋文字與圖示，每階各有 default / hover / active / disabled 四個互動狀態 ( 2026.7.20 拍板新增 )，線條獨立為 stroke，background 再分 surface ( 頁面底層 ) 與 container ( 疊在背景上的卡片、元件、區塊 )

文字

語意化命名取代 Tailwind 原生字級名。✓ 現況即規範 表示程式碼已是這樣，只需抽成 token；★ 新規則 表示需修改程式碼才生效。字級採兩軌制：閱讀內容走 Body1 ( 16px )，密集表格走 Body2 ( 14px )。表格的歸類 ( 2026.7.20 拍板 )：頁面級主資料表屬閱讀內容用 Body1，內嵌於彈窗 / 面板 / 表單的輔助表屬密集表格用 Body2，由 DataTable 元件的 density 一併控制。

互動游標

瀏覽器對 <button> 的預設游標是 default 而不是 pointer（只有 <a> 是 pointer），Tailwind 的 preflight 也沒有補這一條。實務結果是每顆按鈕都要各自寫 cursor-pointer，必然會漏 — demo 曾出現 69 處 @click 只有 1 處寫了游標的情況，使用者難以第一時間辨識何處可點。

已拍板規則 ( 2026.7.20 )：改由 tokens.css 的 @layer base 統一供給，涵蓋 button、[role=button]、[role=tab]、[role=option]、[role=menuitem]、summary、檔案上傳用的 label 與 select。停用態一律排除，游標交還給 not-allowed
元件內不再各自宣告 cursor-pointer。例外：非按鈕語意但可點的元素（帶 @click 的 div、可選取的表格列、可排序的表頭）選擇器管不到，仍須自行加上

狀態色票

狀態色目前定義於 StatusBadge.vue 的對照表，尚未進入 token 系統。這裡如實呈現兩組不同的用色邏輯，正是最需要收斂的部分。

語意狀態 bg-50 / text-700 · 淺底

用於資料狀態、審核結果、敏感度、憑證狀態。四種語意：綠 ( 正常 / 成功 )、琥珀 ( 警示 / 待確認 )、紅 ( 錯誤 / 高風險 )、灰 ( 中性 / 停用 )。

用於紀錄查詢的類別 ( CacheLogCategory )。這組刻意用了 12 種不同色相做辨識，底色階與上方語意色不同 ( -100 而非 -50 )，形成兩套深淺規則 → 審計 04。

08深色模式實際失效中已解決
現況 ( 2026.07.19 拍板 )

已拍板：產品僅支援淺色模式 ( light only )，不提供深色模式。shadcn 元件自帶的 dark: variant 保留不刪，因採 class 策略且全站未掛載 .dark，永遠不會生效、無實際影響；保留可避免日後用 shadcn CLI 更新元件時反覆清理。

處置

此為設計範圍決定而非缺陷，不需程式碼修正。若未來需求變更要支援深色模式，再回設計系統重新開題。
