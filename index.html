01

顏色

Token 架構分三層：Base 層共 13 條純色階 ( 12 條各 10 階 50–900，neutral 多一階 0 純白，共 11 階 )，其中 8 條為核心、5 條僅供分類辨識；Function 層只引用 Base 的階數，不產生新色值；Component 層再收斂一次給實際元件用。需求書的甲方色一律錨定在各色階的 500 作為前景基準，已選取底色 #f0fdfa 自然落在 brand-50。

Layer 1 · Base

純色階，不帶語意

neutral · brand 之外，其餘 11 條依色相排成 red · orange · yellow · green · aqua · sky · blue · indigo · purple · pink · rose，其中 red / orange / yellow / green / blue / aqua 為核心、其餘 5 條僅供分類辨識。這一層只回答「有哪些顏色」，不回答「什麼時候用」

Layer 2 · Function

語意層，只引用 Base

分兩組：中性骨架 ( foreground-primary/secondary、stroke、background-surface/container ) 描述頁面本身；語意色 ( success/warning/danger/info ) 只有 foreground、background、stroke 三個 slot

Layer 3 · Component

元件層，只引用 Function

以實際元件為單位再收斂一次，例如 Tag 元件只有 background 跟 foreground 兩個 slot，指向 Function 層的語意 token

Base 色階 13 色相

甲方需求書六色一律錨定在 500：主操作 #0d9488、已選取底 #f0fdfa ( brand-50 )、警告 #f59e0b、危險 #ef4444、成功 #22c55e、資訊 #3b82f6。

Function 層 · 中性骨架

描述頁面本身的中性色，不含任何語意。foreground 分 primary / secondary 涵蓋文字與圖示，線條獨立為 stroke，background 再分 surface ( 頁面底層 ) 與 container ( 疊在背景上的卡片、元件、區塊 )

文字

語意化命名取代 Tailwind 原生字級名。✓ 現況即規範 表示程式碼已是這樣，只需抽成 token；★ 新規則 表示需修改程式碼才生效。字級採兩軌制：閱讀內容走 Body1 ( 16px )，密集表格走 Body2 ( 14px )。

字體家族 Font Family

中文與英文分別指定，不再依賴系統預設。現況只載入了 Public Sans ( 無中文字符 )，且 fallback 誤寫成無中文的 Noto Sans，導致全站中文實際上是各作業系統的預設字體在顯示。

以下是目前程式碼中相似行為卻樣式分歧、或繞過設計系統的項目。每一條列出現況與收斂建議，供後續逐步調整成正式規範時參考。

01未落實需求書指定的官方色票高
現況

需求書指定六色 ( 主操作 #0d9488 · 已選取底 #f0fdfa · 警告 #f59e0b · 危險 #ef4444 · 成功 #22c55e · 資訊 #3b82f6 )，但 --primary 實際為 #1f8d73、--destructive 為 #e7000b，成功色更以 emerald 取代指定的 green，六色無一在 token 層完整落實。

建議

將六色寫入 token：--primary / --destructive 對齊需求書值，新增 --success / --warning / --info / --selected，頁面一律引用，作為甲方驗收基準。

02兩套顏色系統並行，token 幾乎未被採用高
現況

style.css 定義了完整 token，但頁面幾乎都直接寫 raw Tailwind 色階：文字用 slate-*、品牌用 teal-* 而非 --primary。token 與畫面實際用色是兩回事。

建議

確立 token 為唯一色彩來源，將 raw 色逐步替換為 bg-primary / text-muted-foreground 等語意 class。

03狀態語意色未 token 化高
現況

success / warning / error / info 沒有對應 token，散落在各 feature 以 inline class 手寫。無單一來源可維護。

建議

新增 --success / --warning / --info 等語意 token，狀態元件一律引用。

04共用元件淪為死程式碼高
現況

StatusBadge 與 StatsCard 無任何頁面引用，狀態色改由各頁 inline 重寫，失去單一來源，日後極易走樣。

建議

擇一：讓頁面改用共用元件並補齊 props，或直接刪除避免誤導。收斂後所有狀態色只從一處輸出。

05狀態色混用兩種深淺規則中
現況

語意狀態用 bg-50 / text-700，分類標籤卻用 bg-100 / text-700，兩套底色深度並存於同一個對照表。

建議

把「語意狀態」與「分類辨識」分成兩組獨立 scale，各自統一底色階，用途不混。

06Badge 有兩種形狀中
現況

shadcn 的 Badge 用 rounded-full 膠囊，StatusBadge 卻用 rounded 方角，同為標籤卻兩種外型。

建議

狀態類標籤統一為一種形狀 ( 建議 rounded-full 或 rounded-md 擇一 )，寫入規範。

07文字灰階用了 6 種 slate中
現況

內文從 slate-300 到 slate-900 混用 6 階，但 token 只有 foreground 與 muted-foreground 兩階可對應。

建議

收斂為 3 階：主要 / 次要 / 輔助，分別對應 --foreground、--muted-foreground，必要時補一個中間 token。

08深色模式實際失效中
現況

.dark token 齊備，但頁面用固定 slate-* 色不會隨主題切換，深色模式在實務上是壞的。

建議

頁面改走 token 讓 dark mode 生效；若短期不支援深色，則明確宣告只支援 light 並移除誤導性 token。

094 處寫死 hex 色低
現況

#f3fefa / #6ad7b4 / #f1f5f9 / #f8fafc 直接硬寫在元件內，繞過所有系統。

建議

納入 token 或改用既有色階 ( 如 --muted / --secondary )。

10圓角出現 magic number低
現況

rounded-[4px] 與 rounded-[2px] 與 sm / md / lg / xl / full 並存，未走 radius token。

建議

一律使用 radius token 階層，移除硬寫值。

11邊框用了三種 slate低
現況

border-slate-100 / 200 / 300 三種並用，但 token --border 只有一個值 ( #cbd5e1 )。

建議

統一走 --border，僅特例 ( 如更淺的分隔線 ) 才另訂一個 token。
